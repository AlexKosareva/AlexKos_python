ОТЧЕТ О ДЕФЕКТАХ (DEFECTS REPORT)
Дата: 12.01.2026
Проект: StringUtils Library

---------------------------------------------------------------------------
ID: BUG-01
Заголовок: Метод trim() обрабатывает только один вид пробела (" ")
Приоритет: Средний (Medium)
Описание: Метод жестко ограничен переменной whitespace = " ". Если в начале 
строки стоит символ табуляции (\t) или перенос строки (\n), они не удаляются, 
хотя по логике trim() должны считаться пробельными символами.
Ожидаемый результат: Удаление всех пробельных символов в начале строки.
Фактический результат: Удаляются только обычные пробелы.
---------------------------------------------------------------------------

ID: BUG-02
Заголовок: Метод trim() вызывает ошибку или работает некорректно с None
Приоритет: Высокий (High)
Описание: В методах отсутствует проверка на входной тип данных. При передаче 
None в любой метод программа падает с AttributeError.
Ожидаемый результат: Обработка исключения или валидация типа.
---------------------------------------------------------------------------

ID: BUG-03
Заголовок: Избыточная логика в методе delete_symbol()
Приоритет: Низкий (Low/Optimization)
Описание: Метод сначала вызывает self.contains(), а затем string.replace(). 
Метод replace() в Python сам проверяет наличие подстроки. Двойная проверка 
замедляет работу на больших текстах.
Рекомендация: Удалить условие "if self.contains(...)".
---------------------------------------------------------------------------

ID: BUG-04
Заголовок: Неэффективная реализация trim() через цикл while
Приоритет: Низкий (Optimization)
Описание: Использование цикла while и removeprefix() создает новую строку на 
каждой итерации. При большом количестве пробелов это крайне неэффективно по памяти.
Рекомендация: Использовать стандартный метод .lstrip().
---------------------------------------------------------------------------
